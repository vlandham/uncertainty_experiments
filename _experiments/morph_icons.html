Considers these mock icons representing different event types:

In a dataset, we might have probabilities associated with a particular event, as to the certainty that the event falls into a particular category.

In a map or other view, animation could be used to transition a particular event's icon between all the possible event types it might take.

The amount of time an icon is visible could be proportional to the probability associated with that event type. So more likely event types are displayed for the majority of the time.

Below is a morphing icon with two possible event types - both of equal probabiltiy (50%).

The transition used here is simplistic - but you could imagine more sophisticated transitions being used.

Below is a different morphing event marker with 3 possible event types of different probabilities (70%, 20%, 10%).

One could imagine combining this animated view with a static display of event types and percentages on click or hover. This could be as simple as the icon / name of the event type along with the percent - or a bar representing the probability.
